What is HTML?

HTML is...

• A markup language, which is essential for the creation of web pages, data storage, documentation, etc. Markup languages allow for readability and a clear standard in the use of tags, making it clear to both the systems, and to users who need to understand it.

• Essential for the creation and structure of webpages, allowing developers to easily create and format content, such as headings, paragraphs, links, images, and more.

• NOT a programming language. Although is does lay the foundation for delivering the information on a website from server to the client, it does not control logic, behavior, or perform calculations like a programming language.

• The Web Hypertext Application Technology (WHAT) Working Group's "living standard". This means that HTML 5.0 was designed in a way that has allowed it to be adapted and updated over time, without the need for new versions.

• Under the authority of WHAT Working Group, under an agreement with W3C, which allows them to work on the development of HTML, while avoiding having two standards.

• The core of the World Wide Web. HTML is one of the essential building blocks in the system of the WWW, allowing for the creation of web pages and applications that are accessible to users, all linked together by hyperlinks

• Constantly needing to be evolved as the Web itself develops and changes. This makes it the backbone of the Web, always changing and being built upon to meet its demands.

HTML, once only released as specific versions, is now the Web Hypertext Application Technology Working Group's (WHAT Working Group) “living standard”. This means that while it is a possibility that further versions, such as HTML 6.0, may be released in the future, HTML 5.0 was specifically designed to be continuously updated over time, rather than through traditional updates, allowing it adaptability and flexibility. WHAT Working Group's ability to update HTML over time has been crucial in a time when the Web's advancements and features users require are changing so rapidly. This change from a once static HTML is now living alongside our day-to-day advancements and changes, advancing and changing with them. The World Wide Web Consortium, W3C, was once the sole organization responsible for maintaining HTML and preffered to follow the traditional process of individual versions being released. WHAT Working Group was created in 2004 and wanted to open HTML up to faster development and advancements by introducing HTML as a living standard. For years, W3C and WHAT Working Group worked separately, resulting separate, differing HTML standards. In 2019, an agreement was reached between the two, consolidating HTML development under WHATWG, giving us the HTML living standard that we know today. This agreement ensured a singular standard, while still giving W3C the opportunity to work on the development of HTML. This agreement proved that W3C and WHAT Working Group saw a current and future potential of harm if they continued to have two separate standards, further reinforcing HTML's foundational stability.

Markup languages are essential for the creation and structuring of web pages, data storage, documentation, and so much more. These languages allow for readability and a clear standard in the use of tags, providing clarity to not only the systems reading the code, but also to other users who need to understand it as well. HTML is not to be confused with a programming language such as C, Java, or Python. Although it lays the foundation for delivering the information on the website from the server to the client, it doesn't meet the requirements of a programming language. An article from GeeksforGeeks explains the difference best, “HTML is all about defining the structure and layout of a webpage, focusing on elements like headers, paragraphs, and images. Unlike programming languages, it doesn't control logic, behavior, or perform calculations. Its role is to organize content, not execute operations or interact with data in real time” (GeeksforGeeks 2024). HTML defines the structure and layout of webpages by using specific markup codes, which each represent an HTML element. Some of HTML's basic elements include the element <p>, which defines the beginning of a paragraph, <hr>, or horizontal rule, defining a thematic break, <br>, defining a line break, and <pre>, defining preformatted text. The standard of HTML is an opening, parent <tag> with content intended to be displayed within an ending with the closing, represented by a “/”, parent </tag>. Within these parent tags, there can be child tags, but all child tags must have their ending tag before the parent tag can end.
